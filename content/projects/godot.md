# Godot Engine

## Overview
My engine of choice for interactive multimedia, game development experiments, and educational simulations.

## Why Godot?
- **Open-source** — No licensing fees, full source access
- **Scene/Node system** — Matches how I think about composing interactive systems
- **GDScript** — Python-like syntax for rapid iteration
- **GDExtension** — Drop in C++ modules for performance-critical paths

## Projects

### Procedural Dungeon Generator
2D roguelike using BSP tree room splitting and cellular automata for cave systems. Implemented in GDScript with a C++ GDExtension for the noise generation.

### Physics Sandbox
A learning project exploring Godot's 2D physics engine — rigid bodies, joints, soft-body simulation approximations.

### Portfolio Visualizer Prototype
Early concept for this very portfolio — explored using Godot for the graph visualization before settling on a web-based approach.

## Skills
- GDScript (primary), C++ via GDExtension
- 2D physics, tilemaps, particle systems
- Custom shaders (GLSL-based Godot shader language)
